Imaginary Shift

Solo Programmer

Project Length: 16 Weeks | Team Size: 9 | Engine Used: Unity

What I Did: I created and implemented scripts that were key components for our game. The components include: The main menu, pause menu, end game menus, item pick up, speed boost, and speedometer.

Challenges Overcame: The biggest challenge was being a programmer. I had no programming experience prior to this project, so I had to learn on the fly. I had to think on the fly about what was needed and how to do it. If I couldn’t figure out how to create a script for something, then I needed to find a way to reapproach the problem. The biggest problem for me at the time was getting the speed boost to work properly. After not being able to create a script to make it work, I created an invisible game object that would accelerate the car when the car collided with it.

Follow that Pipe

Game Designer

Project Length: 12 Weeks | Team Size: 12 | Engine Used: Unity

What I Did: I was a designer that focused on creating puzzles for kids to solve. The puzzles would have pipes that would allow the kids to place them down before time runs out. With each level completed, the puzzles would become more difficult little by little.

Challenges Overcame: The biggest challenge for me was making sure these puzzles were easy enough for children to solve. It couldn’t be too easy and it couldn’t be too difficult. I had to make it difficult enough to be challenging, engaging, and fun.
